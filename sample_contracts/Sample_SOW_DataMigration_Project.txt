STATEMENT OF WORK #001

This Statement of Work ("SOW") is entered into as of April 1, 2025,

pursuant to the Master Service Agreement dated March 1, 2025 between:

TechGlobal Solutions Inc. ("Client") and CloudFirst Services LLC ("Provider")

PROJECT: Enterprise Data Migration to AWS Cloud Platform

1. PROJECT OVERVIEW

This SOW describes the scope, deliverables, timeline, and costs for migrating Client's on-premises data infrastructure (approximately 50TB across 12 databases) to Amazon Web Services (AWS), including database migration, ETL pipeline modernization, and data warehouse implementation.

2. SCOPE OF WORK

Phase 1: Assessment & Planning (Weeks 1-4)
- Infrastructure discovery and dependency mapping
- Data classification and sensitivity analysis
- Migration strategy design (lift-and-shift vs re-platforming per workload)
- Risk assessment and mitigation planning
- Detailed project plan with milestones

Phase 2: Environment Setup (Weeks 5-8)
- AWS landing zone configuration (VPC, subnets, security groups)
- IAM roles and policies setup
- Network connectivity (VPN/Direct Connect) configuration
- Monitoring and alerting infrastructure (CloudWatch, SNS)
- Backup and disaster recovery setup

Phase 3: Data Migration (Weeks 9-20)
- Oracle database migration to Amazon RDS (4 databases, ~15TB)
- SQL Server migration to Amazon Aurora PostgreSQL (5 databases, ~20TB)
- MongoDB migration to Amazon DocumentDB (3 databases, ~15TB)
- ETL pipeline migration from Informatica to AWS Glue
- Data validation and integrity verification

Phase 4: Testing & Optimization (Weeks 21-24)
- Performance testing and query optimization
- Security penetration testing
- Disaster recovery testing
- User acceptance testing (UAT)
- Performance benchmarking against baseline

Phase 5: Go-Live & Handover (Weeks 25-26)
- Production cutover with rollback plan
- Knowledge transfer and documentation
- Operations runbook delivery
- 2-week hypercare support

3. DELIVERABLES

7. ASSUMPTIONS AND DEPENDENCIES

- Client will provide timely access to all source systems and environments.
- Source databases are in a consistent and documented state.
- Network bandwidth between on-premises and AWS is adequate (minimum 1 Gbps).
- Client will complete UAT within 10 business days of notification.
- No more than 2 hours of downtime is permitted during production cutover.
- Changes to scope require a written Change Order signed by both Parties.

8. ACCEPTANCE CRITERIA

- All databases migrated with zero data loss verified by checksum validation.
- Query performance within 10% of baseline benchmarks.
- All security controls pass penetration testing.
- 99.9% availability achieved during 2-week hypercare period.
- Deliverables deemed accepted unless Client provides written rejection within 10 business days of delivery.

9. RISKS AND MITIGATIONS

R1: Data volume exceeds estimates → Mitigation: Buffer capacity in timeline; parallel migration streams
R2: Legacy schema incompatibilities → Mitigation: Schema analysis in Phase 1; compatibility testing
R3: Network latency issues → Mitigation: AWS Direct Connect; data transfer optimization
R4: Application compatibility → Mitigation: Early integration testing; rollback procedures
